headless & headless_USERAGENT (Optional)

Here you will find all informations about enable/disable HEADLESS Scrap with your Bot!

If you may ask now what is HEADLESS Scrap then it´s easy to explain. For default we will use Chromium for opening your Browser and you will see your Browser Window.

HEADLESS Scrap instead will open your Browser Window in a silent way which is only running as process in the background but YOU CAN NOT see the Browser Window on your Desktop like usually!



Benefits of HEADLESS Scrap:
• Really Low Hardware Usage
• Faster Website Loading Speed
• More Anonym
• Cookies & Javascript still works! (So you can 1-Time sign-in like usually and the sign-in status will be saved when you restart the Bot)

Negative aspects of HEADLESS Scrap:
• Chrome extensions will not work! (You do not have to remove them! They will just ONLY work when you disable HEADLESS Mode). A lot of websites will include a lot of different third party scripts like Google Analytics or stuff like that. The more external scripts the website will load the longer it will take until the Bot can start to scrap. You can use with NON HEADLESS Browser some extension like ScriptSafe which will block those stuff and increase the loading speed. Because in HEADLESS Mode you can´t use extension some website will may take longer to load!

The benefits from above are possible because a lot of stuff is disabled and chromium doesn´t have to load it when we use HEADLESS!






You have those 2 elements which can be changed at config.json

headless:

"headless":"yes"

You can choose no to disable it!



headless_USERAGENT:

"headless_USERAGENT":"Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/66.0.3171.97 Safari/537.36"

For default the Headless User Agent String includes informations about the HEADLESS Browser. To avoid that a website will ban us for this, you can define at this element a legit User Agent! In this case the User Agent is for Windows 10 - Chrome v66!

Theoretically this User Agent String from above can be used on Windows, MAC & Linux!

Only warranty for User-Agents String that will be listed here at this section!

This timeout will define how long the bot will wait as example until we visit the next page. This timeout is a generall timeout and will be used in scenarios were the bot will do stuff in a loop and bulk open single item pages in a loop.

When you want to use a fixed proxy for the Bot Browser Window that will always be used then you can use the proxy element which you can find inside of the config.json file.

As example:
"proxy":"http://162.243.140.150:8080"



All of my scripts that will download images or other files will use the request module which can be used with a proxy and you can aswell set a custom user-agent.

1. To enable the usage of proxies or custom user-agents for the request module please change those elements to yes:
"enable_proxy_request":"yes"
"enable_useragent_request":"yes"

You can find those elements at file: ./admin/config.json

2. At the folder ./scrap/lib you can find the file proxylist.js & useragent.js where you can insert your proxies or User Agents!

Please notice that the default included proxies in this file are most of the time offline! You must include your own proxies! The same is for the User-Agent list you must include your own! (For default some samples are included!)



IMPORTANT At each new request we will change the proxy & user agent to a random choosen element. If you only want to use 1 specific proxy or user agent instead then you must only enter 1 element at proxylist.js & useragent.js

The bot was written in that way, when ever the request will reach a error(as example when the proxy is not working and the request was not successfully) then the request will automatically restart and choose a new proxy from the list until the request was successfully! In other words the proxy changer will automatically choose a new proxy when the current used one is offline.

When you use FREE public proxies, those proxies will most of the time be offline or not working or no access. You may buy premium proxies instead..
Please notice that the script will use always this http way to connect to a proxy with the request module:
http://12.34.567:89

This means if your proxy needs auth the correct proxy at proxylist.js should be written like this:
http://user:password@12.34.567:89

However the proxy changer was only tested with public proxies like 12.34.567:89! No warranty that it will work aswell with proxies that need´s auth. Please notice that this proxy changer is only a simple tool that will not bypass complex websites protection. For a more specific bypass you must always use a real browser with extensions (WebRTC Anti Leak). Also a lot of proxies are not anonym and they did not hide your real IP! So every website can see that you are still the same user when you did the request!

You can define custom HTTP Headers and more at the options object inside of the downloadIMAGE function which can be found at the top area of the file ./scrap/start.js (No warranty when customer will do changes!)
